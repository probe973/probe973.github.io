Loading Your Dataset in jamovi

Welcome to this guide on loading your dataset into jamovi. For this guide, we will assume your data is in a jamovi file (`.omv`) named omni.omv. Loading your data is the first step to performing any analysis.

How to Load an .omv File into jamovi

jamovi is designed to be very intuitive, and loading data is a simple process using its graphical user interface (GUI).

  1. Open jamovi. You will typically see a blank spreadsheet.
  2. Go to the top-left corner and click on the three horizontal lines menu icon (sometimes called the "hamburger menu").
  3. From the dropdown menu, select Open.
  4. In the "Open" dialog box that appears, you'll see options like "This PC", "Cloud", etc. Select This PC (or wherever your file is stored locally).
  5. Click the Browse button.
  6. Navigate to the folder on your computer where you saved your omni.omv file.
  7. Select omni.omv.
  8. Click Open.

jamovi will now load your data. You should see your variables (columns) and observations (rows) populate the spreadsheet. jamovi is usually very good at detecting the correct data type (e.g., numeric, text) for each variable automatically, but it's always a good idea to quickly inspect your data in the jamovi spreadsheet to ensure everything looks correct.

Inspecting and Changing Variable Types in jamovi

After loading your data, it's crucial to verify that jamovi has correctly identified the data type and measurement level for each of your variables. Incorrect variable types can lead to errors in your analysis or limit the options available to you.

Understanding Variable Types in jamovi:

  • Continuous (ruler icon): For numerical data where values can be any number within a range (e.g., height, age, test scores).
  • Ordinal (bar chart icon): For categorical data where categories have a meaningful order (e.g., Likert scale responses like 'Disagree', 'Neutral', 'Agree').
  • Nominal (three overlapping circles icon): For categorical data where categories have no inherent order (e.g., gender, country, hair color).
  • ID (key icon): Unique identifiers, jamovi will treat these specially and not include them in analyses by default.

How to Inspect and Change Variable Settings:

You can access the variable setup for each column directly from the Data View:

  1. In the jamovi spreadsheet (Data View), double-click on the header of the variable (column name) you wish to inspect or change. For example, double-click on satisfaction_q1.
  2. A "Setup" panel will appear on the right side of your jamovi window for that specific variable.
  3. Variable Type: Under the "Data Type" section, you'll see options like "Continuous", "Integer", "Text".
    • If jamovi has incorrectly identified a continuous variable as text, you can change it here.
    • For categorical variables that are numbers (e.g., 1=Male, 2=Female), jamovi might default them to 'Continuous'. You will need to change them to 'Nominal' or 'Ordinal' here.
  4. Measurement Level: Below the Data Type, you'll see "Measurement Type" with icons for Continuous (ruler), Ordinal (bar chart), and Nominal (three overlapping circles).
    • For example, if your satisfaction_q variables are 1-5 Likert scales, they should be set to Ordinal. If they are currently 'Continuous', change them here.
  5. Levels (for Nominal/Ordinal variables): If you set a variable to Nominal or Ordinal, a "Levels" section will appear. Here you can:
    • Add/Edit Levels: Define the categories and their corresponding values (e.g., Value: `1`, Label: `Strongly Disagree`). This helps jamovi display meaningful labels in your output.
    • Missing Values: Define specific values that represent missing data for this variable.
  6. After making changes, jamovi applies them immediately. You can close the "Setup" panel by clicking outside of it or selecting another variable.

It's good practice to go through each variable, especially categorical ones, to ensure their types and labels are correctly set. This attention to detail at the data setup stage prevents potential issues and ensures accurate analysis later on.
