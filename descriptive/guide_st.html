Descriptive Statistics in Stata

A Guide to Understanding and Summarizing Data

This guide will walk you through the process of obtaining summary statistics and creating relevant graphs for different types of variables using Stata. Understanding descriptive statistics is a foundational skill in data analysis, providing essential insights into the characteristics of your dataset.

Descriptive statistics help us understand the basic features of the data in a study. They provide simple summaries about the sample and the measures, forming the basis of virtually every quantitative analysis.

Before you begin, ensure you have your omni.dta file available. You can enter commands directly into the Stata Command window (the bottom-right pane in Stata by default) or by creating a Do-file (File > New > Do-file Editor). Using a Do-file is highly recommended for saving your commands and making your work reproducible.

In the code examples below, lines starting with `*` are comments. These are notes within the code that explain what the commands are doing. They are very useful for keeping track of your analysis steps in a Do-file. If you are entering commands directly into the Stata Command window, you do not need to copy the * or the comments after it.

First, we need to load the data. We will use the `use` command for this. Before loading your data, ensure your `omni.dta` file is in your Stata working directory. You can set your working directory using the Stata menu: File > Change Working Directory...

use "omni.dta", clear
* This command loads the Stata data file named "omni.dta" into Stata's memory.
* The `clear` option clears any data currently in memory before loading.

Your data is now loaded into Stata. You can refer to variables by their names as they appear in the Data Editor or Variables window.

For more detailed instructions on loading data into Stata, please refer to the Loading Data into Stata Guide.

Part 1: Analyzing Interval/Ratio Variables

Interval/ratio variables are numerical variables where the differences between values are meaningful. For example, a 10-point difference in exam scores means the same regardless of the score range. Ratio variables also have a true zero point (e.g., age, income, test scores, study hours), where zero indicates an absence of the quantity. For these types of variables, we typically examine measures of central tendency (mean, median) and dispersion (standard deviation, variance, range).

1.1 Obtaining Summary Statistics for Interval/Ratio Variables (e.g., final_exam_score)

You can obtain descriptive statistics using the summarize command for basic statistics or the summarize, detail command for a more comprehensive summary.

  1. Using `summarize` (Basic Statistics):

    To get basic statistics like count, mean, standard deviation, minimum, and maximum for a variable like final_exam_score, use the summarize command.

    summarize final_exam_score

    This command will output a table with the number of observations (Obs), Mean, Standard Deviation (Std. Dev.), Minimum (Min), and Maximum (Max) for the specified variable.

  2. Using `summarize, detail` (Comprehensive Summary):

    For a more comprehensive set of descriptive statistics, including N, mean, standard deviation, variance, skewness, kurtosis, and various percentiles (including the median), use the summarize command with the detail option.

    summarize final_exam_score, detail

    This command provides a much richer output, including the mean, standard deviation, variance, skewness, and kurtosis. It also lists the smallest and largest values, along with various percentiles, giving you a detailed picture of the variable's distribution.

1.2 Interpreting Summary Statistics for Interval/Ratio Variables

Here's what each statistic (as provided by summarize, detail) tells us:

  • Percentiles: Values below which a certain percentage of observations fall (e.g., 50th percentile is the median).
  • Smallest/Largest: The minimum and maximum values in your dataset.
  • Obs: The total number of valid observations (N).
  • Mean: The arithmetic average of all values. It is calculated by summing all values and dividing by the number of observations. It is sensitive to extreme values (outliers).
  • Std. Dev.: Standard Deviation. A measure of the typical distance data points are from the mean. A smaller standard deviation indicates data points are clustered closely around the mean, while a larger one indicates more spread or variability in the data.
  • Variance: The average of the squared differences from the mean. It's the square of the standard deviation.
  • Skewness: Indicates the symmetry of the distribution.
    • If Skewness is close to 0, the distribution is relatively symmetrical.
    • If Skewness is positive (>0), the tail of the distribution extends to the right (positive direction). The Mean > Median.
    • If Skewness is negative (<0), the tail extends to the left (negative direction). The Mean < Median.
  • Kurtosis: Indicates the "tailedness" or peakedness of the distribution compared to a normal distribution. Stata's summarize, detail reports raw kurtosis.
    • If Kurtosis is close to 3, it resembles the tailedness and peakedness of a normal distribution (mesokurtic).
    • If Kurtosis is greater than 3 (leptokurtic), it suggests heavier tails and a more peaked distribution, potentially indicating more outliers.
    • If Kurtosis is less than 3 (platykurtic), it suggests lighter tails and a flatter distribution, with fewer extreme values than a normal distribution.

    Note: Different statistical software or packages may use slightly different computational algorithms or default settings for calculating skewness and kurtosis. This can lead to minor variations in the reported numerical values, but the general interpretation of the shape (e.g., positive skew, platykurtic) remains consistent.

1.3 Creating Graphs for Interval/Ratio Variables (Histograms & Box Plots)

Graphs provide powerful visual insights into the distribution of your data, complementing the numerical summaries.

  1. For Histograms:

    Use the histogram command. This command creates a histogram of the specified numerical variable.

    • histogram final_exam_score: This is the basic command to generate a histogram for final_exam_score.
    • normal: (Optional) Adds a normal density curve to the histogram, useful for visual comparison.
    • title("Final Exam Scores"): Sets the main title of the plot.
    • xtitle("Final Exam Score"): Sets the label for the x-axis.
    • ytitle("Density"): Sets the label for the y-axis (Stata's default for histograms is density).

    histogram final_exam_score, normal title("Final Exam Scores") xtitle("Final Exam Score") ytitle("Density")
  2. For Box Plots:

    Use the graph box command. This command creates a box and whisker plot for the specified numerical variable.

    • graph box final_exam_score: This is the basic command to generate a box plot for final_exam_score.
    • title("Final Exam Scores"): Sets the main title of the plot.
    • ytitle("Final Exam Score"): Sets the label for the y-axis.

    graph box final_exam_score, title("Final Exam Scores") ytitle("Final Exam Score")

1.4 Interpreting Graphs for Interval/Ratio Variables

  • Histogram: Displays the shape of the distribution, showing where values are concentrated, identifying peaks (modes), gaps, and overall symmetry or skewness. Each bar represents a range of values (or "bin"), and its height shows the frequency (or density) of scores within that range.
  • Box Plot: Provides a clear visual representation of the "five-number summary": minimum, first quartile (Q1), median (Q2), third quartile (Q3), and maximum. It is particularly effective for identifying outliers.
    • The box itself represents the Interquartile Range (IQR), which contains the middle 50% of your data (between Q1 and Q3).
    • The line inside the box is the median.
    • The "whiskers" typically extend to the most extreme data points within 1.5 times the IQR from the box.
    • Individual points (often shown as dots or asterisks) beyond the whiskers are considered outliers, specifically those falling more than 1.5 times the IQR below Q1 or above Q3. These points represent values unusually far from the rest of the data.

Part 2: Analyzing Categorical Variables

Categorical variables represent types of data which can be divided into groups or categories. This includes nominal variables (categories without inherent order, e.g., 'preferred study method', where 'Flashcards' is not "better" or "more" than 'Summarizing') and ordinal variables (categories with a meaningful order, e.g., satisfaction ratings like 'Low', 'Medium', 'High'). For categorical variables, we focus on frequencies and proportions rather than means and standard deviations, as these mathematical operations are not meaningful for non-numerical categories.

2.1 Obtaining Frequency Tables for Categorical Variables (e.g., preferred_study_method)

  1. To get a frequency table for a categorical variable like preferred_study_method, use the tabulate command. This will give you the counts and percentages for each category.
    2. tabulate preferred_study_method

    This command will output a table showing the frequency (count), percentage, and cumulative percentage for each category of the preferred_study_method variable.

2.2 Interpreting Frequency Tables for Categorical Variables

A frequency table lists each category of your variable and shows how many times it appears in your data (Frequency) and its proportion relative to the total (Percent).

Preferred Study Method Frequency Percent Cumulative
Flashcards 36 36.00 36.00
Practice Tests 31 31.00 67.00
Summarizing 33 33.00 100.00
Total 100 100.00
  • Frequency: The number of participants who selected or fall into each specific category.
  • Percent: The percentage of participants in each category out of the total valid cases (excluding missing values).
  • Cumulative: The running total of percentages, useful for ordinal data but less meaningful for nominal data.

2.3 Creating Graphs for Categorical Variables (Bar Charts)

Graphs provide powerful visual insights into the distribution of your data, complementing the numerical summaries.

  1. To create a bar chart for a categorical variable like preferred_study_method, use the graph bar command. You can choose to display counts or percentages.

    Use the graph bar command. This command creates a bar chart.

    • (count): Specifies that the height of the bars should represent the frequency (count) of observations in each category. You could also use (percent) for percentages.
    • over(preferred_study_method): This option tells Stata to create separate bars for each category of the preferred_study_method variable.
    • title("Preferred Study Methods"): Sets the main title of the plot.
    • ytitle("Frequency"): Sets the label for the y-axis.
    • ylabel(, labsize(small)): Adjusts the size of the y-axis labels.

    graph bar (count), over(preferred_study_method) title("Preferred Study Methods") ytitle("Frequency") ylabel(, labsize(small))

2.4 Interpreting Graphs for Categorical Variables

  • Bar Chart: Each bar represents a distinct category, and its height (or length) corresponds to the frequency or percentage of observations in that category. Bar charts are generally preferred over pie charts for clarity and ease of comparison, especially when comparing frequencies across multiple categories or when categories have similar proportions.

Part 3: Descriptive Statistics by Groups (Splitting Data by Factors)

In data analysis, it is often insightful to understand if a numerical variable's distribution differs across distinct categories of another variable. This is known as "splitting" or "grouping" your data. For example, we might want to see if the final exam scores vary between students who used study support services and those who did not.

3.1 Analyzing Interval/Ratio Variable by a Categorical Factor (e.g., final exam scores by used_study_support_services)

To obtain descriptive statistics for a numerical variable, grouped by a categorical variable, you have two main approaches: using the bysort prefix with summarize, detail for a comprehensive output, or using the tabstat command with its by() option for more selective statistics.

Method 1: Using `bysort` with `summarize, detail`

The bysort prefix in Stata allows you to repeat a command for each unique group within a specified variable. When combined with summarize, detail, it provides a full set of descriptive statistics for your numerical variable, broken down by the categories of your grouping variable.

The general syntax is bysort group_variable: command.

bysort used_study_support_services: summarize final_exam_score, detail

Let's break down what each part of this command does:

  • bysort used_study_support_services:: This prefix tells Stata to sort the data by the used_study_support_services variable first, and then to run the subsequent command (summarize final_exam_score, detail) separately for each unique value (group) of used_study_support_services.
  • summarize final_exam_score, detail: This is the command that is executed for each group. As explained in Part 1.1, the detail option provides a comprehensive summary including mean, standard deviation, skewness, and kurtosis.

This command will produce separate detailed summary tables for final_exam_score for students who 'No' used study support services and those who 'Yes' used study support services.

Method 2: Using `tabstat` with `by()`

The tabstat command is highly flexible and allows you to specify exactly which statistics you want to display for one or more variables, and it can easily group these statistics by a categorical variable using the by() option.

tabstat final_exam_score, by(used_study_support_services) stats(N mean sd median min max skewness kurtosis)

Let's break down what each part of this command does:

  • tabstat final_exam_score: This specifies that you want to calculate statistics for the final_exam_score variable.
  • by(used_study_support_services): This option tells Stata to calculate the statistics separately for each category within the used_study_support_services variable.
  • stats(N mean sd median min max skewness kurtosis): This option specifies exactly which statistics you want to see in the output.
    • N: Number of observations.
    • mean: Mean.
    • sd: Standard deviation.
    • median: Median (50th percentile).
    • min: Minimum value.
    • max: Maximum value.
    • skewness: Skewness.
    • kurtosis: Kurtosis.

This command will produce a single table showing the specified summary statistics for final_exam_score, broken down by used_study_support_services.

Creating Grouped Graphs

Stata also allows you to easily create grouped histograms and box plots using the by() option with the respective graphing commands.

  1. For Grouped Histograms:

    Use the histogram command with the by() option to create separate histograms for each group.

    • histogram final_exam_score, by(used_study_support_services): This creates separate histograms for final_exam_score, one for each category of used_study_support_services.
    • xtitle("Final Exam Score"): Sets the label for the x-axis within each histogram.
    • ytitle("Density"): Sets the label for the y-axis within each histogram.

    histogram final_exam_score, by(used_study_support_services) xtitle("Final Exam Score") ytitle("Density")
  2. For Grouped Box Plots:

    Use the graph box command with the by() option to create separate box plots for each group.

    • graph box final_exam_score, by(used_study_support_services): This creates separate box plots for final_exam_score for each category of used_study_support_services.
    • ytitle("Final Exam Score"): Sets the label for the y-axis.

    graph box final_exam_score, by(used_study_support_services) ytitle("Final Exam Score")

3.2 Interpreting Grouped Descriptive Statistics

By comparing the statistics and plots for each group, you can identify differences in the central tendency, spread, and shape of the numerical variable's distribution across your defined categories.

  • Comparison of Means/Medians: Observe if the average exam scores are different for students who used support services compared to those who did not.
  • Comparison of Standard Deviations: Determine if the variability (consistency) in exam scores is different between the two groups. A larger standard deviation suggests more diverse scores within that group.
  • Comparison of Skewness/Kurtosis: Analyze if the shapes of the exam score distributions differ between the 'Yes' and 'No' groups (e.g., is one group's scores more skewed or more peaked?).
  • Box Plots: Visually compare the median lines, the size of the boxes (IQR), and the presence or absence of outliers between the groups. This provides a quick visual summary of potential group differences.
