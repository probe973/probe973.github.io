Descriptive Statistics in Stata

A Guide to Understanding and Summarising Data using Stata

This guide will walk you through the process of obtaining summary statistics and creating relevant graphs for different types of variables using Stata. Understanding descriptive statistics is a foundational skill in data analysis, providing essential insights into the characteristics of your dataset.

Descriptive statistics help us understand the basic features of the data in a study. They provide simple summaries about the sample and the measures, forming the basis of virtually every quantitative analysis.

Before you begin, ensure you have your omni.dta file available. Throughout this guide, we will use specific variables from your dataset to illustrate the concepts. You will also be prompted to apply the steps to other relevant variables:

  • final_exam_score: Represents the numerical score achieved by students on a final exam.
  • study_hours_per_week: Indicates the number of hours a student reported studying each week.
  • preferred_study_method: Describes a student's chosen study strategy (e.g., 'Flashcards', 'Summarising', 'Practice Tests'). This variable will have already been converted to a numerical nominal format with value labels attached.
  • study_session_group: A categorical variable indicating the type of study session group a student was assigned to (e.g., 'High', 'Medium', 'Low'). This variable will also have already been converted to a numerical nominal format with value labels attached.
  • used_study_support_services: A categorical variable indicating whether a student utilised study support services (Yes/No). This variable will be used to demonstrate splitting analyses by groups, and will also have been converted to a numerical nominal format.

To open your data in Stata:

  1. Launch Stata.
  2. **Menu Driven:** Go to File > Open..., navigate to your omni.dta file, select it, and click Open.
  3. **Command:** In the Command window, type: 
    
use "omni.dta", clear

    The `use` command loads a Stata dataset (`.dta` file) into memory. `"omni.dta"` is the name of your data file. The `clear` option tells Stata to clear any data currently in memory before loading the new dataset. This is useful to prevent errors if you have another dataset open.

Part 1: Analysing Interval/Ratio Variables

Interval/ratio variables are numerical variables where the differences between values are meaningful. For example, a 10-point difference in exam scores means the same regardless of the score range. Ratio variables also have a true zero point (e.g., age, income, test scores, study hours), where zero indicates an absence of the quantity. For these types of variables, we typically examine measures of central tendency (mean, median, mode) and dispersion (standard deviation, variance, range).

1.1 Obtaining Summary Statistics for Interval/Ratio Variables (e.g., final_exam_score)

  1. With your omni.dta file open in Stata.
  2. To get a quick summary of a numerical variable, including N, Mean, Standard Deviation, Min, and Max:
  3. **Menu Driven:** Go to Statistics > Summaries, tables, and tests > Summary and descriptive statistics > Summary statistics. In the dialog box, move final_exam_score to the Variables: box and click OK.
  4. **Command:** In the Command window, type: 
    
summarize final_exam_score

    The `summarize` command provides basic descriptive statistics for numerical variables. `final_exam_score` is the variable you want to summarise.

  5. For more detailed statistics, including Median, Skewness, and Kurtosis:
  6. **Menu Driven:** Go to Statistics > Summaries, tables, and tests > Summary and descriptive statistics > Summary statistics. In the dialog box, move final_exam_score to the Variables: box. Click on the Details tab, then check Display additional statistics (includes skewness and kurtosis). Click OK.
  7. **Command:** In the Command window, type: 
    
summarize final_exam_score, detail

    Adding the `, detail` option to the `summarize` command provides a more extensive table of descriptive statistics, including percentiles (which includes the median), variance, skewness, and kurtosis.

1.2 Interpreting Summary Statistics for Interval/Ratio Variables

Here's what each statistic tells us:

  • Obs (Observations): The total number of valid observations for the variable. Stata automatically handles missing values for these calculations.
  • Mean: The arithmetic average of all values. It is calculated by summing all values and dividing by the number of observations. The mean is sensitive to extreme values (outliers), meaning it can be pulled towards very high or very low scores.
  • Median (50th Percentile): The middle value when data is ordered from smallest to largest. It's the value at which 50% of the data falls below it and 50% above it. The median is less affected by outliers than the mean, making it a good measure of typical value in skewed distributions.
  • Mode: The value that appears most frequently in the dataset. Stata's `summarize, detail` does not directly report the mode, but you can find it using other commands or by inspecting frequency tables for discrete data. For continuous data, it's often more about identifying peaks in a histogram.
  • Std. Dev. (Standard Deviation): A measure of the typical distance data points are from the mean. A smaller standard deviation indicates data points are clustered closely around the mean, while a larger one indicates more spread or variability in the data.
  • Variance: The average of the squared differences from the Mean. It is the square of the standard deviation. It gives a numerical value describing the spread of data points.
  • Smallest & Largest (Min & Max): The smallest and largest values observed in your dataset for that variable. This gives you the range of your data.
  • Skewness: Indicates the symmetry of the distribution.
    • If **Skewness is close to 0**, the distribution is relatively symmetrical (e.g., a bell curve). In symmetrical distributions, the mean, median, and mode are often very close to each other.
    • If **Skewness is positive (>0)**, the tail of the distribution extends to the right (positive direction). This typically means there are more lower values, with a few higher, extreme values pulling the mean to the right of the median. In this case, the Mean > Median.
    • If **Skewness is negative (<0)**, the tail extends to the left (negative direction). This typically means there are more higher values, with a few lower, extreme values pulling the mean to the left of the median. In this case, the Mean < Median.
  • Kurtosis: Indicates the "tailedness" or peakedness of the distribution compared to a normal distribution.
    • If **Kurtosis is positive (>0)** (leptokurtic), it suggests heavier tails and a more peaked distribution, potentially indicating more outliers (extreme values).
    • If **Kurtosis is negative (<0)** (platykurtic), it suggests lighter tails and a flatter distribution, with fewer extreme values than a normal distribution.
    • If **Kurtosis is close to 0** (mesokurtic), it resembles the tailedness and peakedness of a normal distribution.

1.3 Creating Graphs for Interval/Ratio Variables (Histograms & Box Plots)

Graphs provide powerful visual insights into the distribution of your data, complementing the numerical summaries. In Stata, plots will appear in a separate Graph window.

  1. For a Histogram:
  2. **Menu Driven:** Go to Graphics > Histogram. In the dialog box, move final_exam_score to the Variable: box. You can also select the `Normal density plot` checkbox to overlay a normal curve. Click OK.
  3. **Command:** In the Command window, type: 
    
histogram final_exam_score, normal

    The `histogram` command creates a histogram. `final_exam_score` is the variable. The `, normal` option overlays a normal density curve on the histogram, which helps visually assess normality.

  4. For a Box Plot:
  5. **Menu Driven:** Go to Graphics > Box plot. In the dialog box, move final_exam_score to the Variable for y axis: box. Click OK.
  6. **Command:** In the Command window, type: 
    
graph box final_exam_score

    The `graph box` command creates a box plot. `final_exam_score` is the numerical variable for which to create the plot.

    7. Stata will generate these plots in separate Graph windows.

1.4 Interpreting Graphs for Interval/Ratio Variables

  • Histogram: Displays the shape of the distribution, showing where values are concentrated, identifying peaks (modes), gaps, and overall symmetry or skewness. Each bar represents a range of values (or "bin"), and its height shows the frequency of scores within that range.
  • Box Plot: Provides a clear visual representation of the "five-number summary": minimum, first quartile (Q1), median (Q2), third quartile (Q3), and maximum. It is particularly effective for identifying outliers.
    • The box itself represents the **Interquartile Range (IQR)**, which contains the middle 50% of your data (between Q1 and Q3).
    • The line inside the box is the **median**.
    • The "whiskers" typically extend to the most extreme data points within 1.5 times the IQR from the box.
    • Individual points (often shown as dots or asterisks) beyond the whiskers are considered **outliers**, meaning they are unusually far from the rest of the data.

Part 2: Analysing Categorical Variables

Categorical variables represent types of data which can be divided into groups or categories. This includes nominal variables (categories without inherent order, e.g., 'preferred study method') and ordinal variables (categories with a meaningful order, e.g., satisfaction ratings like 'Low', 'Medium', 'High'). For categorical variables, we focus on frequencies and proportions rather than means and standard deviations, as these mathematical operations are not meaningful for non-numerical categories.

2.1 Obtaining Frequency Tables for Categorical Variables (e.g., preferred_study_method)

  1. With your omni.dta file open.
  2. **Menu Driven:** Go to Statistics > Summaries, tables, and tests > Frequency tables > One-way table. In the dialog box, move preferred_study_method to the Categorical variable: box. Click OK.
  3. **Command:** In the Command window, type: 
    
tabulate preferred_study_method

    The `tabulate` command (or `tab` for short) creates one-way frequency tables for categorical variables. It shows the frequency, percentage, and cumulative percentage for each category.

    4. Example Output (for `tabulate preferred_study_method`):

    
preferred_study_method |      Freq.     Percent        Cum.
-----------------------+-----------------------------------
            Flashcards |         36       36.00       36.00
       Practice Tests  |         31       31.00       67.00
          Summarising  |         33       33.00      100.00
-----------------------+-----------------------------------
             Total |        100      100.00

2.2 Interpreting Frequency Tables for Categorical Variables

A frequency table lists each category of your variable and shows how many times it appears in your data (Frequency or Count) and its proportion relative to the total (Percent or Percentage).

Preferred Study Method Frequency Percent Cumulative
Flashcards 36 36.00 36.00
Summarising 33 33.00 69.00
Practice Tests 31 31.00 100.00
  • Frequency: The number of participants who selected or fall into each specific category.
  • Percent: The percentage of participants in each category out of the total valid cases.
  • Cumulative: The running total of percentages. This is more meaningful for ordinal variables where categories have an inherent order.

2.3 Creating Graphs for Categorical Variables (Bar Charts & Pie Charts)

Bar charts and pie charts are effective ways to visually represent the distribution of categorical variables.

  1. For a Bar Chart:
  2. **Menu Driven:** Go to Graphics > Bar chart. Select Simple and then Bars represent: Counts of observations. In the Categories (x axis): tab, move preferred_study_method to the Category variable: box. Click OK.
  3. **Command:** In the Command window, type: 
    
graph bar (count), over(preferred_study_method)

    The `graph bar` command creates a bar chart. `(count)` tells Stata to plot the count of observations for each category. `over(preferred_study_method)` specifies that the bars should be created for each category of `preferred_study_method`.

  4. For a Pie Chart:
  5. **Menu Driven:** Go to Graphics > Pie chart. Select Pie chart for counts of unique values. In the dialog box, move preferred_study_method to the Category variable: box. Click OK.
  6. **Command:** In the Command window, type: 
    
graph pie, over(preferred_study_method)

    The `graph pie` command creates a pie chart. `over(preferred_study_method)` tells Stata to create slices based on the categories of `preferred_study_method`.

    7. Stata will generate these plots in separate Graph windows.

2.4 Interpreting Graphs for Categorical Variables

  • Bar Chart: Each bar represents a distinct category, and its height (or length) corresponds to the frequency or percentage of observations in that category. Bar charts are generally preferred over pie charts for clarity and ease of comparison, especially when comparing frequencies across multiple categories or when categories have similar proportions.
  • Pie Chart: The entire circle represents 100% of the data, and each slice represents a category's proportion of the whole. While visually intuitive for showing parts of a whole, it can be challenging to accurately compare the sizes of slices when categories have similar proportions or when there are many categories.

Part 3: Descriptive Statistics by Groups (Splitting Data by Factors)

In data analysis, it is often insightful to understand if a numerical variable's distribution differs across distinct categories of another variable. This is known as "splitting" or "grouping" your data. For example, we might want to see if the final exam scores vary between students who used study support services and those who did not.

3.1 Analysing Interval/Ratio Variable by a Categorical Factor (e.g., final exam scores by used_study_support_services)

  1. With your omni.dta file open.
  2. To get summary statistics for a numerical variable for each group of a categorical variable:
  3. **Menu Driven:** Go to Statistics > Summaries, tables, and tests > Summary and descriptive statistics > Summary statistics. In the dialog box, move final_exam_score to the Variables: box. Then, move used_study_support_services to the By/If/In: box. Click on the Details tab and check Display additional statistics (includes skewness and kurtosis). Click OK.
  4. **Command:** In the Command window, use the `bysort` prefix: 
    
bysort used_study_support_services: summarize final_exam_score, detail

    The `bysort` prefix tells Stata to perform the subsequent command (here, `summarize final_exam_score, detail`) separately for each unique category of the `used_study_support_services` variable. This generates grouped summary statistics.

    5. Stata will output separate summary tables for each group of 'used study support services'.

  5. To create grouped Box Plots (excellent for visual comparison):
  6. **Menu Driven:** Go to Graphics > Box plot. Move final_exam_score to the Variable for y axis: box. Then, move used_study_support_services to the Categories (x axis): box. Click OK.
  7. **Command:** In the Command window, type: 
    
graph box final_exam_score, over(used_study_support_services)

    The `graph box` command creates box plots. The `over()` option specifies a categorical variable whose categories will be used to create separate box plots, allowing for easy visual comparison between groups.

    8. Stata will generate the grouped box plot in a separate Graph window.

3.2 Interpreting Grouped Descriptive Statistics

By comparing the statistics and plots for each group, you can identify differences in the central tendency, spread, and shape of the numerical variable's distribution across your defined categories.

  • Comparison of Means/Medians: Observe if the average exam scores are different for students who used support services compared to those who did not.
  • Comparison of Standard Deviations: Determine if the variability (consistency) in exam scores is different between the two groups. A larger standard deviation suggests more diverse scores within that group.
  • Comparison of Skewness/Kurtosis: Analyse if the shapes of the exam score distributions differ between the 'Yes' and 'No' groups (e.g., is one group's scores more skewed or more peaked?).
  • Box Plots: Visually compare the median lines, the size of the boxes (IQR), and the presence or absence of outliers between the groups. This provides a quick visual summary of potential group differences.

Part 4: Using Stata Do-Files for Reproducible Analysis

A "do-file" in Stata is a text file containing a sequence of Stata commands. Running a do-file is like typing each command into the Command window one by one, but it allows you to automate your analysis, ensure reproducibility, and keep a clear record of your steps. This is a best practice in quantitative research.

4.1 Creating and Running a Do-File

  1. Open a new Do-file Editor: In Stata, go to Window > Do-file Editor > New Do-file Editor, or click the "New Do-file Editor" icon on the toolbar (looks like a page with a pencil).
  2. Copy Commands: Copy all the Stata commands provided below into the new Do-file Editor window.
  3. Save the Do-file: Go to File > Save As... in the Do-file Editor. Save your do-file with a `.do` extension (e.g., `descriptive_stats_analysis.do`) in the same directory as your `omni.dta` file, or note its path.
  4. Run the Do-file:
    • **To run specific lines:** Select the lines of code you want to run in the Do-file Editor, then click the "Execute (do)" icon (looks like a play button or a folder with a green arrow).
    • **To run the entire file:** Go to Tools > Do in the Do-file Editor menu, or type `do descriptive_stats_analysis.do` (replacing with your file name) in the main Stata Command window and press Enter.

    5. Running the do-file will execute all the commands, producing the same output and graphs as if you typed them manually, but much faster and more reliably.
