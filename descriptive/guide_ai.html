Descriptive Statistics with AI Prompts

A Guide to Understanding and Summarising Data using Artificial Intelligence

Important Warning about AI and Data Analysis:

While AI can be a powerful tool for generating insights and assisting with data analysis, it's crucial to understand its limitations. **AI models can make mistakes, "hallucinate" data, or provide incorrect interpretations, especially if the input is unclear, ambiguous, or if it lacks real-world context.**

Always verify the output of an AI model against your raw data and your understanding of statistical principles. Treat AI as a helpful assistant, not as a replacement for critical thinking and human oversight in data analysis. The examples in this guide assume you will input real data into the AI entity.

This guide will walk you through the process of obtaining summary statistics and creating relevant graphs for different types of variables by formulating effective prompts for an AI entity. Understanding descriptive statistics is a foundational skill in data analysis, providing essential insights into the characteristics of your dataset.

Descriptive statistics help us understand the basic features of the data in a study. They provide simple summaries about the sample and the measures, forming the basis of virtually every quantitative analysis.

Before you begin, **please ensure you have uploaded your omni.csv file to the AI entity** (if your AI platform supports file uploads). If not, you will need to describe your data structure and provide relevant snippets directly in your prompts as demonstrated in the examples. Throughout this guide, we will refer to variables from this dataset to illustrate effective prompting strategies. You will be encouraged to adapt these prompts for your own specific data.

  • final_exam_score: Represents the numerical score achieved by students on a final exam.
  • study_hours_per_week: Indicates the number of hours a student reported studying each week.
  • preferred_study_method: Describes a student's chosen study strategy (e.g., 'Flashcards', 'Summarising', 'Practice Tests').
  • study_session_group: A categorical variable indicating the type of study session group a student was assigned to (e.g., 'High', 'Medium', 'Low').
  • used_study_support_services: A categorical variable indicating whether a student utilised study support services (Yes/No).

Part 1: Analysing Interval/Ratio Variables with AI

Interval/ratio variables are numerical variables where the differences between values are meaningful. For these, we examine measures of central tendency (mean, median, mode) and dispersion (standard deviation, variance, range), along with distribution shape (skewness, kurtosis).

1.1 Obtaining Summary Statistics for Interval/Ratio Variables (e.g., final_exam_score)

When asking an AI for descriptive statistics, be clear about the variable, its type, and what statistics you need. Refer to your uploaded `omni.csv` file.

Example Prompt:


From the `omni.csv` dataset I uploaded, please calculate the mean, median, mode, standard deviation, variance, minimum, maximum, skewness, and kurtosis for the `final_exam_score` variable. Also, provide a brief interpretation of each statistic.

**Explanation:** This prompt explicitly states the dataset (`omni.csv`), the variable name (`final_exam_score`), and a comprehensive list of desired statistics. It also asks for an "interpretation," encouraging the AI to provide context for the numbers.

1.2 Interpreting Summary Statistics for Interval/Ratio Variables (AI Output)

Based on a prompt like the one above, an AI might provide output similar to what you would get from statistical software. You would then interpret these just as you would with any other analysis. The AI's explanation can serve as a guide, but always cross-reference with your own knowledge.

  • Mean: The average score.
  • Median: The middle score when arranged in order.
  • Mode: The most frequent score(s).
  • Standard Deviation: How spread out the scores are from the mean.
  • Variance: The average of the squared differences from the mean (standard deviation squared).
  • Minimum & Maximum: The lowest and highest scores.
  • Skewness: Indicates if the scores are symmetrically distributed or if they lean to one side (tail). Positive skew means a tail to the right (more lower scores), negative to the left (more higher scores).
  • Kurtosis: Describes the "tailedness" and "peakedness" of the distribution. High kurtosis means more extreme outliers and a sharper peak, low means flatter and fewer outliers.

1.3 Creating Graphs for Interval/Ratio Variables with AI (Histograms & Box Plots)

You can ask AI to describe or even generate code for visualisations. The capability of AIs to provide actual plots varies. Some AI tools, including those integrated into productivity suites (e.g., certain versions of Microsoft Copilot in Office) or dedicated data analysis AI interfaces, can directly generate and display the visual plots within their output.

Example Prompt for AIs that Directly Render Plots:


Using the `omni.csv` dataset, please generate a histogram and a box plot for the `final_exam_score` variable. Ensure appropriate labels and titles are included in the plots.

**Explanation:** This prompt directly asks the AI to generate the visual plots. Use this type of prompt if your AI tool is known to have integrated plotting capabilities.

For AI models that are not capable of rendering the visual plots (e.g., Gemini), you can ask for Python or R code to generate these plots yourself.

Example Prompt for AIs that Provide Plotting Code:


Using the `omni.csv` dataset, please describe what a histogram of the `final_exam_score` variable would look like and what it would tell me about the data's distribution. Also, describe what a box plot would show, focusing on central tendency, spread, and any potential outliers. Additionally, provide Python code to generate these plots from the `final_exam_score` column in a pandas DataFrame.

**Explanation:** This prompt asks for both a description of what the plots reveal and, crucially, asks for executable code. You can then copy and run this code in a separate programming environment like Google Colab, Jupyter Notebooks, or a Python/R IDE to generate the actual visualisations yourself. This allows you to verify the AI's descriptive interpretation and engage in hands-on coding. It also specifies the format of the data (pandas DataFrame) for clarity if you intend to run the code.

1.4 Interpreting Graphs for Interval/Ratio Variables (AI Output & Your Visualisation)

Once you generate the plots using the AI's code (or if the AI gives a textual description of the plot):

  • Histogram: Visually assess the shape of the distribution, showing where values are concentrated, identifying peaks (modes), gaps, and overall symmetry or skewness.
  • Box Plot: Identify the median line (middle of the box), the range of the central 50% of data (the box itself), the extent of the "whiskers" (typical data range), and any individual points beyond the whiskers (potential outliers).

Reflection Questions & Tasks:

  2. Ask the AI to describe what a histogram and a box plot of the `study_hours_per_week` variable from `omni.csv` would look like, noting any potential skewness or outliers.
  3. Task: Use a prompt to ask the AI to summarise the `study_hours_per_week` variable from `omni.csv`. Then, based on its output, describe the distribution's shape, spread, and central tendency in your own words.

Suggested Solutions for Reflection Questions (Part 1):

  1. If the mean is notably higher than the median for `study_hours_per_week`, the AI should explain that this suggests a **positive skew** in the distribution, meaning there's a tail extending to higher values (e.g., a few students studying very long hours pulling the mean up). If the mean is lower than the median, it indicates **negative skew**.
  2. For `study_hours_per_week`, the AI should describe a histogram that might show a slight positive skew (more bars on the left, with a tail to the right) and a box plot that might show a few individual points beyond the upper whisker, indicating outliers of very high study hours.

Part 2: Analysing Categorical Variables with AI

For categorical variables, we focus on frequencies and proportions, as mathematical operations like means are not meaningful. Refer to your uploaded `omni.csv` file and the variable names.

2.1 Obtaining Frequency Tables for Categorical Variables (e.g., preferred_study_method)

When asking AI for frequencies, refer to the categorical variable in your uploaded file.

Example Prompt:


From the `omni.csv` dataset, please create a frequency table showing the count and percentage for each category of the `preferred_study_method` variable.

**Explanation:** This prompt clearly refers to the dataset and the specific categorical variable. It explicitly asks for both counts and percentages in the frequency table.

2.2 Interpreting Frequency Tables for Categorical Variables (AI Output)

The AI's output should list each category with its corresponding frequency (count) and percentage. You would interpret this similar to software output:

Preferred Study Method Count % of Total
Flashcards 36 36.0%
Summarising 33 33.0%
Practice Tests 31 31.0%
  • Count: The number of instances for each category.
  • %: The proportion of the total sample that falls into each category.

2.3 Creating Graphs for Categorical Variables with AI (Bar Charts & Pie Charts)

You can ask the AI to describe these charts or generate code for them, similar to numerical variables. The capability of AIs to provide actual plots varies. Some AI tools, including those integrated into productivity suites (e.g., certain versions of Microsoft Copilot in Office) or dedicated data analysis AI interfaces, can directly generate and display the visual plots within their output.

Example Prompt for AIs that Directly Render Plots:


Using the `omni.csv` dataset, please generate a bar chart and a pie chart for the `preferred_study_method` variable. Ensure appropriate labels and titles are included in the plots.

**Explanation:** This prompt directly asks the AI to generate the visual plots. Use this type of prompt if your AI tool is known to have integrated plotting capabilities.

For AI models that are not capable of rendering the visual plots (e.g., Gemini), you can ask for Python or R code to generate these plots yourself.

Example Prompt for AIs that Provide Plotting Code:


Using the `omni.csv` dataset, please describe what a bar chart for the `preferred_study_method` variable would look like, emphasising which method is most popular. Also, describe what a pie chart would convey for the same variable. Additionally, provide Python code to generate both a bar chart and a pie chart from the `preferred_study_method` column in a pandas DataFrame.

**Explanation:** This prompt reiterates the dataset and variable name, and explicitly asks for descriptions and code for both types of charts, specifying what visual insights are expected. You can then copy and run this code in a separate programming environment like Google Colab, Jupyter Notebooks, or a Python/R IDE to generate the actual visualisations yourself. This allows you to verify the AI's descriptive interpretation and engage in hands-on coding.

2.4 Interpreting Graphs for Categorical Variables (AI Output & Your Visualisation)

  • Bar Chart: Look for the tallest bar to identify the most frequent category. Compare bar heights to understand relative popularity.
  • Pie Chart: Observe the size of each slice relative to the whole circle. Larger slices represent larger proportions. Best for showing "parts of a whole."

Part 3: Descriptive Statistics by Groups with AI

When you want to compare a numerical variable across different groups of a categorical variable, you need to clearly structure your prompt to define both the numerical variable and the grouping variable from your uploaded `omni.csv` file.

3.1 Analysing Interval/Ratio Variable by a Categorical Factor (e.g., final exam scores by used_study_support_services)

Provide the AI with clear instructions to analyse data from the specified columns in your uploaded file.

Example Prompt:


Using the `omni.csv` dataset, please calculate the mean, median, and standard deviation for the `final_exam_score` variable, grouped by the `used_study_support_services` variable. Then, briefly compare and interpret the differences between the 'Yes' and 'No' groups based on these statistics.

**Explanation:** This prompt explicitly refers to the dataset and both the numerical and categorical variables. It clearly asks for grouped statistics and a comparative interpretation.

3.2 Interpreting Grouped Descriptive Statistics (AI Output)

The AI should provide separate statistics for each group, allowing for direct comparison. This output will be similar to what you would find in statistical software.

  • Comparison of Means/Medians: Look for differences in average scores between groups.
  • Comparison of Standard Deviations: Assess if one group's scores are more consistent (smaller SD) or more varied (larger SD) than another.
  • Comparison of Skewness/Kurtosis: Analyse if the shapes of the exam score distributions differ between the 'Yes' and 'No' groups (e.g., is one group's scores more skewed or more peaked?).
  • Box Plots: Visually compare the median lines, the size of the boxes (IQR), and the presence or absence of outliers between the groups. This provides a quick visual summary of potential group differences.

Part 4: Best Practices for Prompting AI for Data Analysis

To get the most accurate and useful results from an AI when performing descriptive statistics, follow these best practices:

  1. Be Specific and Clear: Explicitly state what you want. Instead of "Summarise this data," say "From the `omni.csv` dataset, calculate the mean, median, and standard deviation for the `final_exam_score` variable."
  2. Provide Context: Tell the AI about the type of variables if it's not self-evident from the data (e.g., "The `final_exam_score` is a numerical variable," or "The `preferred_study_method` is a categorical variable.").
  3. Ensure Data Access: Clearly state that the data is from an uploaded file (e.g., "Using the `omni.csv` dataset..."). If your AI platform does not support file uploads, you must paste relevant data subsets directly into the prompt.
  4. Ask for Interpretation: Don't just ask for numbers; ask the AI to "interpret" or "explain the implications" of the results. This pushes the AI to provide qualitative insights.
  5. Request Specific Output Format: Ask for results in a table, a specific format (e.g., "provide the output in a markdown table"), or as executable code (e.g., "provide Python code for the plots from a pandas DataFrame").
  6. Iterate and Refine: If the initial response isn't what you need, ask follow-up questions. "Can you explain the skewness in simpler terms?" or "Now, compare the results for the 'Yes' and 'No' groups more directly."
  7. Cross-Verify: This is the most critical step. Always manually check the AI's calculations, compare its interpretations to your own knowledge, and run any generated code to visually inspect plots. Do not blindly trust AI output.
